Introduction

Studying all of the code examples in the repository is critical for exam preparation, but using the VSCode debugger in class is optional. If you prefer, you can just watch the instructor and/or use the web server and JavaScript debugger of your choice.
Setting up the VS Code Debugger
  1. Install Live Server for VS Code so that you can run a web server on port 5500 at the click of a button.
  2. In VS Code Settings, enable the "Debug: Allow Breakpoints Everywhere" setting.
  3. In VS Code Settings, set the "Debug: Open Debug" option to openOnSessionStart.

Using the VS Code Debugger
  1. Click View->Run.
  2. Make sure that "Launch Chrome against localhost" is selected next to the green triangle in the top left corner.
  3. Click the green triangle. It will open a separate instance of Chrome. Use the dropdown at the top to select the appropriate demo.
    Note: if you receive an error that says "Configured debug type 'pwa-chrome' is not supported", try updating your version of VS code.
  4. Use the Debugger controls to step through the code as desired, examine variable values, etc.
